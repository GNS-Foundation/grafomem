The Platform

Three Pillars of Governed AI

Memory, Governance, and Attestation — the complete stack for compliant AI agent deployment.

🧠

Verifiable Memory

GMP-conformant memory stores with PostgreSQL + pgvector. Every write, retrieve, delete, and supersede is audit-logged with full provenance.

  • 384-dim HNSW vector search
  • Ed25519 content signatures
  • Multi-store per tenant
  • Semantic + keyword retrieval
🛡️

Policy-as-Code Governance

PDP/PEP architecture with a separated Policy Engine and Governance Gateway. Rate limits, PII guards, and human-in-the-loop escalation — enforced before every inference.

  • Stateless policy evaluation
  • Append-only evidence logs
  • PII detection + redaction
  • HITL escalation gates
🔗

Cryptographic Attestation

Every agent step produces a hash-chained execution receipt. Every decision is Ed25519-signed. Every deletion generates a GDPR erasure certificate. Tamper-evident by construction.

  • BLAKE2b-256 hash chains
  • Deterministic replay engine
  • Signed erasure certificates
  • One-click regulatory reports

Under The Hood

7-Layer Governance Stack

Built on the open GMP specification (MIT, v0.2.0). Every layer adds a governance guarantee that compounds into compliance-grade AI.

Layer 1 — Memory
GMP-conformant stores · PostgreSQL + pgvector · HNSW 384-dim · Write / Retrieve / Delete / Supersede
Layer 2 — Decision Trail
Ed25519-signed inference logs · Full context capture · Retrieved facts + prompts + outputs
Layer 3 — Erasure Proof
GDPR Article 17 certificates · Content hash attestation · Signed deletion proof
Layer 4 — Governance
PDP/PEP architecture · Policy Engine (stateless) · Evidence Collector (append-only) · HITL gates
Layer 5 — Reports
One-click EU AI Act / GDPR / DORA compliance packages · Risk assessments · Audit bundles
Layer 6 — Orchestrator
8-step governed execution loop · Sequential / Supervisor / Round-robin · BYOM LLM registry
